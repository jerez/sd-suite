# Security Policy

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

If you believe you have found a security vulnerability in this repository,
please report it privately through GitHub's security reporting features when
available, or contact the maintainer privately through GitHub.

When reporting, include:

- a description of the issue
- affected files, features, or workflows
- reproduction steps or proof of concept
- any known impact

## Supported Versions

Security fixes are handled on a best-effort basis for the current mainline
development state of the repository.

## Expectations

Security reports will be reviewed, but response timelines and patch
availability are not guaranteed.
